# Nest CLI plugins under SWC (Swagger / GraphQL)

The Nest CLI's `@nestjs/swagger` / `@nestjs/graphql` plugins are TypeScript transformers that
auto-generate OpenAPI / GraphQL metadata from your types — so you don't hand-write `@ApiProperty()`
on every DTO field. SWC can't run those transformers inline. nestkit bridges the gap with
`@mgvdev/nestkit-nest-plugin`, which runs Nest's `PluginMetadataGenerator` at build time to emit a
`metadata.ts` you load at bootstrap.

## Setup

Install the plugin bridge and the Nest plugin you use:

```bash
npm i -D @mgvdev/nestkit-nest-plugin @nestjs/cli
npm i @nestjs/swagger        # and/or @nestjs/graphql
```

Enable it in the app's `nestkit.json`:

```jsonc
{ "type": "app", "entry": "src/main.ts", "nestPlugins": ["swagger"] }
```

Load the generated metadata at bootstrap:

```ts
import { DocumentBuilder, SwaggerModule } from '@nestjs/swagger'
import metadata from './metadata' // generated by nestkit build
import { AppModule } from './app.module'

async function bootstrap() {
  const app = await NestFactory.create(AppModule)
  await SwaggerModule.loadPluginMetadata(metadata) // <-- before createDocument
  const doc = SwaggerModule.createDocument(app, new DocumentBuilder().build())
  SwaggerModule.setup('docs', app, doc)
  await app.listen(3000)
}
```

## How it works

On `nestkit build` (and each `nestkit dev` rebuild), for any `app` with `nestPlugins`, nestkit runs
the plugin's `ReadonlyVisitor` through `PluginMetadataGenerator` and writes `metadata.ts` into the
app's `src/`. SWC then compiles it like any other file. DTOs get their schema from your TypeScript
types — no `@ApiProperty()` required:

```ts
export class CatDto {
  name!: string
  age!: number
}
// → OpenAPI: { name: { type: string }, age: { type: number }, required: [name, age] }
```

## Notes

- Add `src/metadata.ts` to `.gitignore` — it's a build artifact.
- Requires `@nestjs/cli` (for `PluginMetadataGenerator`) resolvable from the workspace root, plus the
  plugin package (`@nestjs/swagger` / `@nestjs/graphql`).
- Uses TypeScript's classic compiler API — TypeScript **5.x / 6.x** (not 7.x).
